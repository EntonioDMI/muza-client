# apps/desktop/src/shell/QueuePanel.tsx

Очередь (UX-доводка 2026-07-11): секции «История / Сейчас / Далее»,
удаление/перестановка/очистка хвоста, «Сохранить как плейлист», «К текущему».
Закрытая панель НЕ живёт в DOM (unmount) — Tab не попадает в невидимые кнопки.

---

Два компонента в файле: `QueueRow` (одна строка трека) и `QueuePanel`
(секции + шапка) — оба вызывают свой собственный `useT()`.

**i18n (2026-07-14, T34a, эпик W5):** строки извлечены в `dialogs.queue.*`
через `const { t } = useT();`. Особенности:
- aria-label панели и заголовок «Очередь» переиспользуют `player.queue`
  (идентичный текст); счётчик приклеивается отдельным ключом
  `dialogs.queue.countSuffix` (" · {count}").
- ⚠️ `history.map((t, i) => ...)` и `upNext.map((t, i) => ...)` используют
  `t` как имя ПЕРЕМЕННОЙ ТРЕКА (затеняет переводчик `t` из `useT()`) —
  это уже существовавший код, не тронут; переводчик внутри этих `.map`
  callback'ов не вызывается (только `track={t}` — прокидывается трек),
  так что затенение безопасно, но не путай при будущих правках.
- «Свернуть»/«Показать (N)» для истории — НЕ переиспользуют
  `listeningMode.minimize` (тот же русский текст «Свернуть», но другой
  контекст — сворачивание секции истории, не режима прослушивания); заведён
  свой `dialogs.queue.collapse`, чтобы EN-текст («Collapse», не «Minimize»)
  читался осмысленно именно для этого действия.
