Вход в совместный плейлист по инвайт-коду (Stage 7). Код выдаёт владелец
через `CollabDialog.tsx` («Совместный доступ»).

---

**i18n (2026-07-14, T34a, эпик W5):** строки извлечены в
`dialogs.joinPlaylist.*` через `const { t } = useT();`. «Код короче 4
символов…» переиспользует общий `dialogs.codeTooShort` — идентичный текст
уже был в `JamDialog.tsx`. Кнопка «Отмена» — `common.cancel`.

**Подпись про бэкенд (2026-07-15).** Проп `apiHost: string | null` — в дев-сборке
под полем ввода печатается `dialogs.devBackend` («Dev build: only codes from
localhost:8000 will work»), в проде `null` → подписи нет. Считает `App.tsx`
через `lib/devApiHost.ts` (чистая функция, `isDev` параметром), прокидывает
`Player`. Причина: инвайт-код живёт в Postgres КОНКРЕТНОГО сервера и с прода на
localhost не переезжает, а дев- и released-сборки визуально идентичны — 15.07
владелец из-за этого завёл баг на исправную фичу (его пользователь выдал код на
проде, дев-сборка искала в локальной БД). Тест — `JoinPlaylistDialog.test.tsx`
(рендерится без `LanguageProvider` → ждёт английские строки).

⚠️ `onJoined(playlist)` вызывается ВНУТРИ `try` рядом с `api.joinPlaylist` —
исключение оттуда покажется пользователю как ошибка ВХОДА ПО КОДУ. Сегодня
безопасно (колбэк синхронный, асинхронное внутри — за `void`), но при
переписывании колбэка на `await` это станет ловушкой.

Серверная половина той же правки — текст 404 больше не утверждает «проверь
символы» (`muza-server/src/me/collab.controller.ts`).
