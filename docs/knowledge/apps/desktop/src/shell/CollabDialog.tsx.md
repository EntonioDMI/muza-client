# apps/desktop/src/shell/CollabDialog.tsx

«Совместный доступ» плейлиста (Stage 7): диалог с инвайт-кодом (владелец —
создать/скопировать/отозвать) и списком участников (кик у владельца, «Покинуть
плейлист» у участника). Рендерится из `views/PlaylistView.tsx`.

---

Внутри `<LanguageProvider>` (потребитель `PlaylistView.tsx` уже вызывает
`useT()`) — компонент такой же.

**i18n (2026-07-14, T34a, эпик W5):** все строки извлечены в
`dialogs.collab.*` через `const { t } = useT();`. Особенности:
- Собранный из двух частей JSX-текст «Совместный плейлист пользователя
  **X**. Ты можешь добавлять и убирать треки.» разбит на три ключа
  (`sharedPrefix`/`sharedByUser`/`sharedCanEdit`), чтобы сохранить `<b>`
  вокруг имени владельца — `t()` возвращает строку, JSX-разметку внутри неё
  вставить нельзя.
- `владелец`/`owner` (фолбэк имени владельца + текст бейджа роли) —
  один ключ `dialogs.collab.ownerFallback`, переиспользуется в обоих местах
  (совпадает текстуально).
- Общие для нескольких shell-диалогов ключи (не в `collab.*`, а прямо в
  `dialogs.*`): `close` («Закрыть»), `copyFailed` («Не удалось
  скопировать»), `copyCode` («Скопировать код» — и Tooltip, и IconButton
  label зовут один и тот же ключ).
- `memberRow(...)` — локальная функция-рендерер внутри компонента (не
  отдельный компонент), замыкает `t` из области видимости компонента.
