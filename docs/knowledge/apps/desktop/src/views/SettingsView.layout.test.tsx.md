# apps/desktop/src/views/SettingsView.layout.test.tsx

Сторож контракта раскладки настроек (t5-settings-space, 2026-07-16): скелет
DOM, на который завязаны `>`-селекторы `SettingsView.layout.css`, aria-связки
рельса, сброс скролла панели и структурная половина EN/RU-инварианта.
Создано: 2026-07-16.

---

- **Что честно тестируемо в jsdom, а что нет**: ширины/высоты/`@container` —
  дело CSS-лэйаута, которого в jsdom не существует, их проверяет визуальная
  приёмка. Тестируется то, что переживает jsdom: вложенность узлов (контракт
  селекторов layout.css), атрибуты aria, поведение эффекта скролла и
  идентичность структуры между языками.
- **Рендер-харнесс**: минимальные пропсы + `api: {} as unknown as MuzaApi` —
  безопасно, потому что с `serverSession=false`, дефолтной вкладкой
  «Внешний вид» и без `intent` ни один mount-эффект до api не дотягивается
  (все гейтятся tab/sub/serverSession; `listInstalled` за `isTauri()` → `[]`).
  Без LanguageProvider useT() фолбэкает на EN — прецедент
  `PlaylistView.test.tsx`.
- ⚠️ **`role="tab"` есть не только у рельса**: сегментные `Tabs` из `@muza/ui`
  внутри панели («Тема», «Язык интерфейса»…) тоже tablist/tab — счёт «10
  пунктов» обязан скоупиться `.muza-settings-nav [role='tab']`, иначе
  ловится 17.
- **Тест сброса скролла** сначала проверяет ПРЕДПОСЫЛКУ (`pane.scrollTop =
  400; expect(400)`) — jsdom без лэйаута не клампит присвоение к нулю, но
  если это поведение jsdom изменится, ассерт «после клика 0» стал бы пустым;
  guard делает деградацию громкой. Именно поэтому в SettingsView.tsx выбран
  `scrollTop = 0`, а не `scrollTo()` (тот в jsdom не реализован).
- **EN/RU-тест** сравнивает id пунктов и число детей рельса между рендерами
  без провайдера (EN) и с `LanguageProvider lang="ru"`, плюс контроль, что
  заголовки реально различаются (иначе сравнение было бы двух одинаковых
  рендеров).
