# apps/desktop/src/app.css

Глобальный App-уровневый CSS (импорт в `main.tsx`): ресет html/body/#root,
keyframes фона «Анимированный» (T15), запрет выделения текста и БАЗОВЫЙ каркас
вертикальной навигации настроек (`.muza-settings*`). Эта заметка заведена в
ветке t5-settings-space (2026-07-16) и покрывает только связь с редизайном
раскладки настроек; полная история файла — в знании корневой рабочей папки
(`docs/knowledge/muza-client/apps/desktop/src/app.css.md`, из worktree
недоступна). Обновлено: 2026-07-16.

---

- **Комментарии в самом файле — первоисточник** по механике каркаса настроек:
  почему вертикальная навигация, почему `@container` по ширине панели (а не
  `@media` по окну), почему `container-type` безопасен для fixed-потомков
  (замеры в шапке блока), ловушка с анимацией `.muza-view` в headless.
- ⚠️ **С 2026-07-16 ГАБАРИТЫ каркаса** (`grid-template-columns` у
  `.muza-settings__cols`, `max-width: 680px` у `__pane`, `--r-md` у рельса,
  `align-items: start`) **перекрыты** более специфичными правилами из
  `views/SettingsView.layout.css` — числа в комментариях app.css для габаритов
  устарели, править размеры надо там (см. его knowledge). В app.css остаются
  живыми: механика `@container` (порог 600px, скрытие подписей/заголовка,
  сужение гэпа, центровка пунктов в 48px-рельсе), стекло плашки
  (`--glass-sidebar`/`--bf-zone`), метрики пункта 48px/sp-4 и типографика.
  Маркер об этом — комментарий, ДОБАВЛЕННЫЙ В КОНЕЦ app.css (append в конец —
  сознательно: файл параллельно правят другие ветки, вставка в середину
  ловила бы конфликты).
- Правки этой ветки в app.css — ТОЛЬКО этот маркер-комментарий; ни одно
  правило не менялось.
