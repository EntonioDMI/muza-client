## Зачем пенёк, а не массовая правка импортов

Модуль i18n импортируют **45 файлов** десктопа строкой `from "../i18n"` (или
`"../../i18n"`). Переписать их все на `"@muza/app/i18n"` — 45 файлов диффа в
этапе, который к ним не имеет отношения: Э0 доказывает, что React-пакет
проходит через оба бандлера, а не переставляет импорты по всему приложению.

Пенёк даёт **нулевой дифф у потребителей**: `export * from "@muza/app/i18n"`
резолвит те же имена, TypeScript видит те же типы (включая `TranslationKey`),
рантайм-стоимости нет — сборщик схлопывает реэкспорт.

Приём работает без потерь только потому, что модуль экспортирует ИСКЛЮЧИТЕЛЬНО
именованные символы: ни `export default`, ни side-effects. Был бы дефолтный
экспорт — пенёк пришлось бы писать руками, и бесплатным он бы уже не был.

## Срок жизни

**Пеньки сносятся в Э9** — последним косметическим коммитом марафона, когда 45
импортов перепишутся одним махом, отдельно от содержательных правок и без
риска утонуть в шуме ревью. До тех пор пенёк бесплатен.

**Новый код импортирует напрямую из `"@muza/app/i18n"`** — не через этот путь.
Пенёк обслуживает существующие импорты, а не приглашает заводить новые.
