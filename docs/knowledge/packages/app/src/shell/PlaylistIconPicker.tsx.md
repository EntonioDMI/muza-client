# packages/app/src/shell/PlaylistIconPicker.tsx

Пикер иконки плейлиста (T47b): диалог с сеткой всех 38 иконок манифеста
`@muza/core`. Клик сразу применяет (без отдельного «Сохранить») — вызывающий
код зовёт `api.setPlaylistIcon` и закрывает диалог сам. С Э0 веб-паритета
(2026-07-15) — **общий компонент десктопа и веба**, сведён из двух копий.
Обновлено: 2026-07-16 (скроллбар сетки → `scrollbarWidth: "thin"`).

---

Два компонента в файле: `IconSwatch` (одна плитка-иконка) и
`PlaylistIconPicker` (диалог с сеткой) — оба вызывают свой `useT()`.

**i18n (2026-07-14, T34a, эпик W5):** строки извлечены в
`dialogs.iconPicker.*` через `const { t } = useT();`. `iconAria` содержит
параметр `{id}` — id иконки (слаг манифеста, латиница) подставляется как
есть в оба языка.

## Э0 веб-паритета (2026-07-15): переезд + сведение двух копий

Файл переехал `apps/desktop/src/shell/PlaylistIconPicker.tsx` →
`packages/app/src/shell/PlaylistIconPicker.tsx`. В отличие от i18n (чистый
`R100`) это **не переименование, а сведение**: у веба была своя копия
(`apps/web/src/components/PlaylistIconPicker.tsx`), обе удалены, общая — здесь.
Копии успели разъехаться: десктопная 90 строк с `useT()` и инлайн-стилями,
веб-копия 51 строка с CSS-классом `.icon-swatch` и **захардкоженным русским**
прямо в JSX.

**Для этого документа главное:** T34a пережил сведение без потерь — `useT()`
остался в обоих компонентах, ключи `dialogs.iconPicker.{title,iconAria}` те же,
параметр `{id}` подставляется как раньше. Победила именно десктопная сторона
по строкам: перевода у веба не было вообще, и сведение его туда принесло.
Обратное направление тоже есть — тач-таргеты ≥44px взяты из веб-копии.

Практическое следствие: словарь `dialogs.iconPicker.*` теперь читают ДВА
приложения. Русский заголовок в вебе держится на `LanguageProvider lang="ru"`
(`apps/web/src/providers.tsx`, добавлен тем же этапом) — без провайдера
`useT()` молча фолбэкнулся бы на `DEFAULT_LANG="en"`, и в русском интерфейсе
веба появился бы английский диалог.

⚠️ `"use client"` в файле НЕТ намеренно (в веб-копии была) — пакет не содержит
директив вообще, клиентскую границу держат приложения. Директива внутри пакета
уронила бы сборку Vite.

## Скроллбар сетки (2026-07-16)
Контейнер сетки (`maxHeight: min(60vh, 420px)` + `overflowY: auto`) получил
`scrollbarWidth: "thin"` — до этого был единственным скроллом приложения с
системным «толстым» скроллбаром. Паттерн взят у выпадающей панели `Select`
(`packages/ui/src/components/core/Select.jsx:147`); прятать полосу совсем
(`"none"`, как main/Sidebar) нельзя — она единственный намёк, что иконок
больше, чем видно. `::-webkit-scrollbar` не вариант: стили компонента инлайновые
(самодостаточность пакета), псевдоэлементы в `style=` не записываются.
Тест №6 в `PlaylistIconPicker.test.tsx` пиннит это поведение.

## ⚠️ «Иконка = арт трека» — заблокирована сервером (2026-07-16)
Хотелка «ПКМ по треку → его обложка становится иконкой плейлиста» без серверной
работы нереализуема: `PATCH /me/playlists/:id/icon` валидирует
`/^pi-\d{2}$/` + `@Length(1,16)` (`muza-server/src/me/dto.ts:9,114-119`),
колонка `Playlist.icon` — `VarChar(16)` (`prisma/schema.prisma:268`), и
`playlistIconSrc` (`@muza/core`) отдаёт `null` на всё вне манифеста. Не хакать
клиентским обходом: иконка не доедет ни до БД, ни до рендера, ни до веба.
