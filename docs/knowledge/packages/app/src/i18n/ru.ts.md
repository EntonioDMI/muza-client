# packages/app/src/i18n/ru.ts

Русский словарь (эпик W5 i18n) — типизирован как `typeof en` (см.
`en.ts.md`), форма ОБЯЗАНА совпадать 1:1 с английским словарём (tsc ловит
расхождение). Подмешивает `ru.views` и `ru.media`. С Э0 веб-паритета
(2026-07-15) живёт в `@muza/app`. Обновлено: 2026-07-15 (переезд).

---

**T34a (2026-07-14, эпик W5):** зеркально добавлены зоны `dialogs`, `mini`,
`plugins` — см. `docs/knowledge/packages/app/src/i18n/en.ts.md` за полным
описанием состава (структура/ключи идентичны, здесь только русские значения).

Русские значения переносились ДОСЛОВНО из исходных захардкоженных строк
диалогов (не перефразировались) — единственная цель правки была извлечь
существующий русский текст в словарь, а не изменить его. Английские
значения — новый естественный перевод, сверенный с уже существующим
вокабуляром `en.ts` (Home/Search/Favorites/Library/Jam/Queue/Share и т.п.,
синонимы не вводились).

## Э0 веб-паритета (2026-07-15): переезд в `@muza/app`

Переехал `apps/desktop/src/i18n/ru.ts` → `packages/app/src/i18n/ru.ts` вместе со
всем модулем i18n, **без правок содержимого** (git: `R100` — чистое
переименование, 0 строк диффа). Обоснование переезда и приём «пенёк» —
`docs/knowledge/apps/desktop/src/i18n/index.tsx.md`.

Практический эффект **именно для русского словаря** сильнее, чем для
английского: `ru.ts` стал словарём, который реально показывается в ДВУХ
приложениях. Веб монтирует `LanguageProvider lang="ru"`
(`apps/web/src/providers.tsx`), и общий `PlaylistIconPicker` берёт
`dialogs.iconPicker.{title,iconAria}` отсюда — а раньше веб-копия пикера
держала те же строки ЗАХАРДКОЖЕННЫМИ в JSX. То есть русский текст, который в
вебе жил в разметке, теперь живёт здесь, в одном месте с десктопным.

⚠️ `lang="ru"` в вебе прибит НАМЕРЕННО (не читается из prefs): весь остальной
UI веба сегодня русский хардкодом, переключатель показал бы половину
интерфейса переведённой. Оживёт в Э8 — до тех пор `ru.ts` для веба не
«язык по выбору», а единственный.
