# Плагины Muza

Справочник для разработчиков плагинов: манифест, API, extension points, публикация в маркетплейс. Живые примеры — в [`examples/`](./examples).

## 1. Два уровня

Плагин Muza живёт на одном из двух уровней — выбирается правами в `manifest.json`:

- **Уровень 1 — песочница (по умолчанию).** Код плагина исполняется в изолированном `<iframe sandbox="allow-scripts">` (opaque origin: без DOM хоста, без cookies/storage приложения, без сети в обход `Muza.Net`). Хочешь повлиять на приложение — вызываешь `window.Muza.*`: структурный API поверх плеера/очереди/библиотеки/UI/хранилища/сети/событий. Хост исполняет изменения **от имени** плагина; сам код плагина в хост никогда не попадает.
- **Уровень 2 — «Полный доступ»** (право `app:full-access`). Код плагина исполняется **в окне самого приложения** — сырой DOM, React-дерево, `window.__TAURI__`. Может менять всё — и поэтому идёт с громким согласием пользователя при установке и обязательной премодерацией перед публикацией в маркетплейс. См. §8.

Главный принцип обоих уровней: **право на каждый вызов**. Плагин объявляет нужные права в `permissions`; вызов метода без соответствующего права возвращает ошибку `denied` вместо тихого отказа или похода в обход.

## 2. Быстрый старт

Минимальный плагин — папка с двумя файлами:

```
my-plugin/
  manifest.json
  index.js
```

`manifest.json`:

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "api_version": 1,
  "description": "Что делает плагин, одной фразой",
  "author": "твой ник",
  "entry": "index.js",
  "permissions": ["ui.tab"]
}
```

`index.js`:

```js
(function () {
  Muza.UI.toast("Привет из my-plugin!", "sparkles");
})();
```

**Установка из файла:** заархивируй содержимое папки (не саму папку — файлы должны лежать в корне архива) в `.muzaplugin` (обычный zip с другим расширением) и поставь через Настройки → Расширения → «Установить из файла». Приложение покажет список запрошенных прав — подтверди, чтобы установить.

Полные рабочие примеры с кнопкой в баре, вкладкой, пунктом меню и уровнем 2 — в [`examples/`](./examples).

## 3. Манифест

`manifest.json` в корне пакета.

| поле | тип | обяз. | смысл |
|---|---|---|---|
| `id` | string, `^[a-z0-9](?:[a-z0-9-]{1,38})[a-z0-9]$` | да | уникальный id (латиница/цифры/дефис, 3–40 символов, без дефиса по краям); путь установки — `$APPDATA/plugins/<id>/` |
| `name` | string ≤ 60 | да | отображаемое имя |
| `version` | semver (`x.y.z`) | да | версия плагина |
| `api_version` | int, сейчас `1` | да | мажор API Muza; несовпадение мажора — плагин не загрузится |
| `description` | string ≤ 200 | да | строка для карточки |
| `author` | string ≤ 32 | да | автор (в маркетплейсе перетирается ником публикатора) |
| `entry` | string, относительный путь (`index.js`) | да | точка входа JS, без `..`, без ведущего `/`/`\`, без Windows drive-letter |
| `permissions` | string[] (§4) | да (можно `[]`) | запрашиваемые права; `app:full-access` переводит плагин на уровень 2 |
| `contributes` | объект (§6) | нет | декларативные extension points уровня 1 |
| `net_allow` | string[] (хосты) | нет | домены, разрешённые для права `net` |
| `min_app_version` | semver | нет | не загружать на более старой Muza |

`contributes` — всё опционально, каждый блок требует своего права (§4, §6):

```json
{
  "tabs":       [{ "id": "translator", "title": "Перевод", "icon": "languages" }],
  "barButtons": [{ "id": "translate",  "title": "Перевод строки", "icon": "languages" }],
  "navItems":   [{ "id": "translator", "title": "Перевод", "icon": "languages" }],
  "menus": {
    "track":        [{ "id": "translate-title", "title": "Перевести название" }],
    "catalogTrack": [{ "id": "translate-title", "title": "Перевести название" }],
    "playlist":     [{ "id": "export", "title": "Экспортировать" }]
  },
  "panel":   { "id": "lyrics-plus", "title": "Текст+" },
  "overlay": { "id": "visualizer" },
  "css":     "theme.css",
  "strings": "strings.json"
}
```

Пример полного манифеста уровня 1:

```json
{
  "id": "sync-translator",
  "name": "Синхро-переводчик",
  "version": "1.2.0",
  "api_version": 1,
  "description": "Перевод строк текста на лету, вторая строка под оригиналом",
  "author": "polyglot",
  "entry": "index.js",
  "permissions": ["player.read", "events.track", "ui.tab", "ui.slots", "storage", "net"],
  "net_allow": ["translate.googleapis.com"],
  "contributes": { "tabs": [{ "id": "translator", "title": "Перевод", "icon": "languages" }] }
}
```

Уровень 2 — то же самое, но `"permissions": ["app:full-access"]` (другие права не нужны — в хост-контексте доступно всё; `contributes` можно оставить для честной декларации, хотя хост его для full-access не использует).

`icon` в `contributes.*` — имя иконки из [lucide](https://lucide.dev/icons) (kebab-case, как в `lucide-react`), например `"languages"`, `"audio-waveform"`, `"sparkles"`.

## 4. Права

Уровень 1 — запрашивай только то, что реально нужно (модалка согласия при установке выделяет «опасные» права отдельно):

| право | даёт |
|---|---|
| `player.read` | читать состояние плеера, текущий трек, очередь |
| `player.control` | play/pause/next/prev/seek/громкость/скорость |
| `player.queue` | менять очередь (enqueue/remove/reorder/clear/playTrack) |
| `library.read` | читать плейлисты и «Любимое» |
| `library.write` ⚠️ | менять плейлисты и «Любимое» |
| `ui.tab` | открывать свою вкладку, показывать тосты |
| `ui.slots` | добавлять кнопки в бар, пункты сайдбара/меню, панель, оверлей |
| `ui.theme` ⚠️ | менять оформление приложения (CSS) |
| `strings` | подменять текст интерфейса (см. §7 — пока `not_yet`) |
| `storage` | хранить свои данные на диске (до 1 МБ) |
| `net` ⚠️ | ходить в сеть на хосты из `net_allow` |
| `events.track` | события смены трека и позиции воспроизведения |
| `events.playback` | события воспроизведения и очереди |
| `events.library` | события библиотеки (лайки, плейлисты) |
| `events.ui` | события интерфейса (смена вкладки, темы) |

Уровень 2 — **`app:full-access`** ⚠️⚠️ — поглощает все остальные права. Чем платишь:
- при установке — отдельный громкий экран согласия (чекбокс «понимаю риск» + задержка кнопки, кнопка отказа — дефолтная);
- в маркетплейсе — бейдж «Полный доступ» и **обязательная ручная премодерация** админом перед тем, как плагин станет виден кому-либо, кроме автора (§10);
- выключение плагина **не выгружает** уже исполненный код — realm окна нельзя выгрузить без перезапуска приложения (§8).

## 5. API v1

Пространства доступны как `window.Muza.<ns>.<method>(...)` внутри песочницы уровня 1. Все методы асинхронные (возвращают Promise); вызов без права — реджект с `err.code === "denied"`.

### Player

| метод | право | что делает |
|---|---|---|
| `Player.getState()` | `player.read` | текущее состояние плеера |
| `Player.getCurrentTrack()` | `player.read` | текущий трек (метаданные, без URL источника) |
| `Player.getQueue()` | `player.read` | очередь (метаданные) |
| `Player.play()` / `pause()` / `next()` / `prev()` | `player.control` | транспорт |
| `Player.seek(sec)` | `player.control` | перемотка |
| `Player.setVolume(v)` / `setRate(r)` | `player.control` | громкость/скорость |
| `Player.enqueue(trackIds, pos?)` | `player.queue` | добавить в очередь |
| `Player.removeFromQueue(pos)` | `player.queue` | убрать из очереди |
| `Player.reorderQueue(from, to)` | `player.queue` | переставить |
| `Player.clearQueue()` | `player.queue` | очистить очередь |
| `Player.playTrack(trackId)` | `player.queue` | запустить конкретный трек |

### Library

| метод | право |
|---|---|
| `Library.getPlaylists()` / `getPlaylistTracks(id)` / `getFavorites()` | `library.read` |
| `Library.createPlaylist(name)` | `library.write` |
| `Library.addToPlaylist(id, trackIds)` / `removeFromPlaylist(id, trackIds)` | `library.write` |
| `Library.like(trackId)` / `unlike(trackId)` | `library.write` |

### UI

| метод | право | что делает |
|---|---|---|
| `UI.toast(text, kind?)` | `ui.tab` | показать тост (`kind` — имя иконки lucide, опционально) |
| `UI.openTab(tabId)` | `ui.tab` | открыть свою вкладку из `contributes.tabs` |
| `UI.openPanel()` / `openOverlay()` / `closeSurface()` | `ui.slots` | открыть/закрыть свою панель/оверлей |
| `UI.setBadge(slotId, text)` | `ui.slots` | бейдж на своей кнопке бара/пункте |
| `UI.setBarButtonState(id, { icon?, active? })` | `ui.slots` | обновить вид кнопки бара |
| `UI.applyCss(css)` / `removeCss()` | `ui.theme` | инъекция/снятие CSS в хост (§7) |

### Storage / Net

| метод | право | детали |
|---|---|---|
| `Storage.get(key)` / `set(key, value)` / `remove(key)` / `keys()` | `storage` | KV на диске, неймспейс — твой `id`, квота 1 МБ (`quota` при превышении) |
| `Net.fetch(url, init?)` | `net` | только хосты из `net_allow`, только `https://`, без cookies; сеть плагина возможна ТОЛЬКО через этот метод |

### Events

```js
Muza.Events.on("track:change", function (track) { ... });
Muza.Events.off("track:change", handler);
```

| тип события | право |
|---|---|
| `track:change`, `position` (~1/с) | `events.track` |
| `playback:state`, `queue:change` | `events.playback` |
| `like:change`, `playlist:change` | `events.library` |
| `view:change`, `theme:change` | `events.ui` |
| `slot:click` | не требует отдельного права — это твой собственный слот (§6) |

## 6. Extension points

Объявляются в `contributes`, право `ui.slots` (кроме `tabs`, которым хватает `ui.tab`):

- **`barButtons`** — кнопка в плеер-баре.
- **`navItems`** — пункт сайдбара. `id` должен совпадать с `id` записи в `tabs`, чтобы клик открывал именно эту вкладку.
- **`tabs`** — полноценная своя вкладка (контент — тот же фрейм плагина, развёрнутый на весь экран содержимого).
- **`menus.track` / `menus.catalogTrack` / `menus.playlist`** — пункт в контекстном меню трека / плейлиста, дописывается в конец существующего меню. `track` и `catalogTrack` теперь ведут в ОДНО И ТО ЖЕ меню трека и равнозначны: разделение осталось с тех пор, когда рядом с настоящим меню жило меню демо-трека из макета Stage 1 (удалён в 0.1.2). Оба слота работают, объявляй любой; в новых плагинах бери `track`.
- **`panel`** — своя боковая панель (как «Сейчас играет»).
- **`overlay`** — полноэкранный слой поверх приложения (например, визуализатор).

Все клики по твоим кнопкам/пунктам меню приходят ОДНИМ событием `slot:click` с `{ slotId, payload? }` — `payload` есть только у пунктов меню (метаданные трека/плейлиста, на который кликнули):

```js
Muza.Events.on("slot:click", function (data) {
  if (data.slotId === "translate") {
    // кнопка бара — payload нет
  }
  if (data.slotId === "translate-title" && data.payload) {
    // пункт меню трека — data.payload = { id, title, artist }
  }
});
```

У плагина ОДИН постоянный фрейм на всю сессию (не пересоздаётся при переключении поверхности) — он всегда живой (получает `slot:click` и события фоном), а видимость (скрыт / вкладка / панель / оверлей) переключает хост через CSS. Открыть свою вкладку/панель/оверлей программно — `UI.openTab(tabId)` / `UI.openPanel()` / `UI.openOverlay()`.

**Композиция:** твои `barButtons`/`navItems` встраиваются в те же настройки композиции, что и родные элементы (Настройки → Внешний вид → Бар/Сайдбар) — пользователь может их переставлять и выключать, пока плагин установлен и включён.

## 7. CSS-темизация и строки

**CSS** (право `ui.theme`) — два пути: статический файл `contributes.css` (путь относительно пакета) и/или динамический `UI.applyCss(css)` / `UI.removeCss()`. Хост рендерит твой CSS отдельным `<style data-plugin="<id>">` в корне приложения, после пользовательской кастомной темы. Выключил плагин — `<style>` снимается.

CSS проходит скан (клиент — при установке, сервер — при публикации): запрещены `@import`, `expression()`, `javascript:`, `<script`, `behavior:`, `-moz-binding:`, backslash-обфускация и `url()` с чем-либо, кроме `data:` (внешние адреса — утечка).

**Строки** (право `strings`) — `contributes.strings` (JSON `{ "ключ": "текст" }`) и/или `Strings.override(map)` / `Strings.reset()`. ⚠️ Право и метод уже объявлены, но подмена ПОКА не реализована по-настоящему (единый строковый слой интерфейса появится в отдельном эпике i18n) — вызов сейчас честно возвращает ошибку `not_yet`, а не тихо ничего не делает.

## 8. Уровень 2 — «Полный доступ»

Когда 6 не хватает — например, нужно физически перестроить DOM в месте, до которого структурный API не дотягивается. Прежде чем тянуться к `app:full-access`, перечитай §6 — почти всё «изменить приложение» покрывается уровнем 1 без компромисса безопасности.

**Механика.** Твой `entry` исполняется одной инъекцией через `WebviewWindow::eval` (нативная привилегия хоста — не через `eval()`/`new Function()` в JS, эти пути заблокированы CSP приложения и на уровне 2 тоже) при старте приложения и при каждом включении плагина. Код обёрнут в try/catch хостом; в замыкании доступен `__muzaPlugin` — `{ id, name, version, reportError(message) }`. Внешний try/catch ловит только СИНХРОННЫЙ throw во время загрузки — для асинхронного кода (промисы, таймеры) заворачивай сам и зови `__muzaPlugin.reportError(...)` вручную, иначе ошибка потеряется молча.

**Ответственность.** Ты получаешь то же самое, что имеет само приложение: сырой DOM, React-дерево через DOM, `window.__TAURI__`. Это значит: доступ к серверной сессии пользователя, ко всем его настройкам, ко всем данным окна. Пиши так, будто пишешь код самой Muza — потому что фактически так и есть.

**Правила ответственного использования** (см. также комментарии в [`examples/full-access-demo`](./examples/full-access-demo)):
1. Не трогай данные пользователя (localStorage/cookies/сеть), которые не являются прямой целью твоего плагина.
2. Делай только ОБРАТИМЫЕ и ЗАМЕТНЫЕ изменения — пользователь должен уметь опознать, что сделал именно твой плагин.
3. Не патчи глобальные прототипы, не перехватывай события верхнего уровня — это ломает остальное приложение и соседние full-access-плагины непредсказуемо.
4. Не делай ничего, что нельзя было бы объяснить одним предложением в описании манифеста.

**Ограничения честно:**
- Выключение плагина НЕ выгружает уже исполненный код (realm окна нельзя выгрузить на лету) — приложение предложит перезапуск сразу после выключения; до перезапуска код продолжает работать.
- Скан кода (§10) прогоняется и для уровня 2, но НЕ является защитой сам по себе — защита это согласие пользователя + премодерация автора + доверие. Скан ловит только явное (`eval`, `new Function`, `require`, `import`).
- Watchdog уровня 1 неприменим — зависание твоего кода означает зависание всего приложения.

## 9. Сеть и хранилище

- **Storage** — `Storage.get/set/remove/keys`, KV на диске в своём неймспейсе (`$APPDATA/plugins/.storage/<id>.json`), квота **1 МБ** — превышение возвращает `quota`.
- **Net** — `Net.fetch(url, init?)`: только хосты из `net_allow` манифеста, только `https://`, ответы приватным/loopback/link-local/CGNAT адресам блокируются (SSRF-фильтр), тело ответа ограничено 5 МБ. Прямой `fetch`/`XMLHttpRequest`/`WebSocket` из кода плагина уровня 1 запрещён publish-сканом маркетплейса — вся сеть только через `Muza.Net`.

## 10. Публикация в маркетплейс

1. Собери плагин: `manifest.json` + `entry` одним JS-файлом (маркетплейс — однофайловый payload, `contributes.css`/`contributes.strings` — отдельными полями). Суммарный размер payload ≤ **128 КБ**.
2. `POST /market/plugins` с телом `{ manifest, code, css?, strings? }` (тот же формат, что использует клиент — см. пример запроса ниже).
3. Скан (тот же, что при локальной установке, + серверные лимиты): манифест — Zod-схема; JS — AST-скан (acorn): запрещены `eval`, `Function`/`new Function`, `require`, `import`/`export` (плагин однофайловый); для уровня 1 дополнительно запрещены `fetch`/`XMLHttpRequest`/`WebSocket`/`sendBeacon` (сеть только через `Muza.Net`) и обращения к `window.parent`/`window.top`/`self.parent`/`self.top` (побег из песочницы) — эти два запрета на уровень 2 не распространяются, там код и так в хост-контексте; CSS — тот же скан, что в §7; лимиты на длину строк и глубину AST (анти-обфускация).
4. **Уровень 1** — прошёл скан → виден в витрине сразу. **Уровень 2** (`app:full-access`) → публикация уходит в статус «на модерации» (`pending`), видна только тебе, пока админ вручную не одобрит (`POST /market/plugins/:id/approve`) — премодерация full-access обязательна, без неё плагин никому, кроме автора, не виден. Обновление уже одобренного full-access-плагина снова уходит на премодерацию (код изменился).
5. Публикация — до 5 плагинов в час на пользователя; жалобы (`POST /market/plugins/:id/report`) — до 10 в час; 5 уникальных жалоб автоматически скрывают плагин из витрины до решения модератора.
6. Повторная публикация с тем же `manifest.id` от того же автора — обновление записи; `manifest.id` глобально уникален — попытка опубликовать чужой существующий id отклоняется.

Пример публикации (замени `$TOKEN` и путь к файлам):

```bash
curl -X POST http://localhost:8000/api/market/plugins \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d @- <<'JSON'
{
  "manifest": { "...": "manifest.json целиком" },
  "code": "...содержимое index.js одной строкой..."
}
JSON
```

## 11. Ограничения и антипаттерны

- **Нет доступа к URL/токенам источников звука** ни на одном уровне честного пути (уровень 1 — по дизайну песочницы; на уровне 2 технически видно то же, что видит приложение, но это не повод — см. §8 «ответственность»). Метаданные (название/артист/обложка) доступны, сырые ссылки — нет.
- **`eval`/`new Function` не работают** нигде: CSP песочницы уровня 1 (`script-src 'nonce-...'`, без `unsafe-eval`) и CSP приложения в целом (`script-src 'self'`) их блокируют; уровень 2 исполняется через нативную инъекцию Rust, а не через JS-`eval` хоста, так что этот путь для плагина тоже закрыт.
- **Таймауты и watchdog** (уровень 1): вызов метода без ответа 5 секунд — `timeout`; фрейм, не прошедший рукопожатие `ready` за 3 секунды или не отвечающий на пинг дважды подряд — снимается из DOM (плагин помечается «зависшим», пользователь может перезапустить).
- **Выгрузка уровня 2 — только рестартом приложения** (§8) — не проектируй плагин так, будто выключение мгновенно останавливает его код.
- **Плагин однофайловый** для маркетплейса — `import`/`require`/динамические импорты запрещены; сторонние библиотеки нужно вкомпилировать в `entry` целиком (bundler на твоей стороне, до публикации).
