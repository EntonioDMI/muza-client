## Anti-patterns verdict

Muza не выглядит как шаблонный AI-интерфейс. У неё есть музыкальная идентичность, постоянный транспорт, сильная пространственная модель и осмысленная работа с обложкой и текстом. Автоматический detector завершился с `0 findings`.

Главный риск иной: prototype slop. Некоторые знакомые поверхности выглядят законченными, но не дают ожидаемого стандартного контроля: очередь read-only, часть фильтров декоративна, многие действия доступны только мышью, а ошибки иногда маскируются красивым fallback.

## Общее впечатление

Muza уже убедительна как музыкальная сцена, но ещё не всегда убедительна как ежедневный десктопный инструмент. Самая крупная возможность — сделать поведение таким же зрелым, как внешний вид: честные состояния, управляемая очередь, полная клавиатурная модель и меньше публичного roadmap в Settings.

## Что работает

- Композиция sidebar + content + Now Playing + player bar создаёт устойчивую пространственную память.
- Player bar покрывает реальный музыкальный домен: транспорт, seek, volume, EQ, lyrics, queue, speed, repeat, sleep timer.
- Импорт плейлиста честно показывает долгую работу, matched/total и ненайденные позиции.
- Пустые состояния Любимого и локальной библиотеки объясняют следующий шаг.
- На поддерживаемом минимуме 1024x700 нет горизонтального overflow; контраст видимых экранов прошёл пороги.

## Приоритетные проблемы

### [P1] Интерактивные компоненты не имеют цельной клавиатурной и screen-reader модели

`Tile` и `TrackRow` кликабельны мышью, но не являются полноценными keyboard targets; `Slider` и EQ `Fader` не реализуют Arrow/Home/End; `Menu` не имеет roving focus; `Dialog` не управляет переносом, trap и возвратом фокуса; поля Settings/Login зависят от placeholder и подавляют outline. Это ломает доступность и противоречит power-user позиционированию.

### [P1] Закрытая QueuePanel остаётся в accessibility tree

Панель скрывается opacity/transform/pointer-events, но её кнопки остаются достижимы через Tab. Пользователь может попасть в невидимый интерфейс.

### [P1] Очередь выглядит полноценной, но почти не управляется

Нет remove, clear, reorder, history/up-next, save as playlist и undo. Для музыкального приложения это одна из самых нативных рабочих поверхностей, поэтому несоответствие ожиданиям особенно заметно.

### [P1] Главная скрывает правду о loading/offline/error

Начальная загрузка выглядит пустой областью; сбой без snapshot превращается в демо-полки без пометки. Пользователь считает неперсональные данные рекомендациями и не понимает, что система работает в fallback-режиме.

### [P1] Settings смешивают рабочую панель и публичный roadmap

Десять равноправных вкладок, две строки навигации и множество пунктов «позже» увеличивают когнитивную нагрузку и делают продукт визуально незавершённым. Нужны 5-6 верхних групп, поиск настроек и скрытие неработающих функций.

## Persona red flags

### Alex, power user

- Есть глобальные hotkeys, но нет batch actions и полноценного keyboard workflow в queue/EQ/menu.
- Speed и sleep timer требуют циклических кликов по presets.
- Hover-only actions замедляют работу со списками.

### Jordan, первый пользователь

- Chips на Home выглядят как фильтры, но не меняют выдачу.
- Демо-fallback не подписан.
- Термины «источники», «добыча», `-14 LUFS`, версии и бесконечное радио требуют контекстного объяснения.

### Sam, keyboard/screen-reader user

- Закрытая очередь остаётся фокусируемой.
- Fields Settings/Login не имеют надёжного accessible name/focus state.
- Tile/TrackRow/Slider/Fader/Menu/Dialog имеют неполную доступность.
- Многие цели 28-36px меньше рекомендуемых 44px.

### Riley, stress tester

- Empty/error/offline смешиваются в один Home fallback.
- Live search может молча не сработать.
- Queue не имеет empty state.
- Неверные списки значений в Settings отбрасываются без объяснения.

## Полный UX-бэклог по поверхностям

### Home и рекомендации

- Заменить blank loading на skeleton.
- Развести состояния: новый аккаунт, нет истории, offline snapshot, сервер недоступен, демо.
- Подписывать stale/offline/demo данные и давать Retry.
- Сделать chips настоящими фильтрами или убрать их до реализации.
- Вынести «не нравится» и «не рекомендовать артиста» ближе к треку.

### Player bar

- Разделить controls по приоритетам; вторичные перенести в overflow на узком окне.
- Убрать фиксированную зависимость seek от ширины 480px; ввести compact tiers.
- Сделать обложку настоящей кнопкой с accessible name.
- Добавить keyboard semantics и озвучиваемые значения seek/volume.
- Сделать активные repeat/sleep/speed/EQ состояния заметнее, чем небольшая иконка.

### Queue

- Добавить current, up next, history.
- Добавить remove, clear, reorder, save as playlist, undo.
- Добавить empty state, sticky current и «перейти к текущему».
- При открытии переносить фокус внутрь, при закрытии возвращать на trigger.
- При `open=false` использовать unmount/hidden/inert.

### Search

- Показывать recent queries и понятный pre-search state.
- Не глотать ошибку live search; сохранять запрос и давать Retry.
- Для анонима добавить CTA «Войти», а не только объясняющий текст.
- Развести «ничего не найдено» и «источники недоступны».

### Library и playlists

- Не использовать Play-иконку, если действие только открывает плейлист.
- Убрать demo-поведение, где разные Tile запускают один фиксированный трек.
- Добавить sort/filter, multi-select и batch actions.
- Показывать offline/download/source availability на уровне списка без перегруза.
- Дать явную обработку недоступных локальных треков.

### Import

- Добавить `aria-live`/progressbar semantics.
- Позволить свернуть долгий импорт в background task.
- После partial match дать действия «Найти сейчас», «Пропустить», «Экспортировать список».
- Проверить, не теряется ли итоговый отчёт при автоматическом переходе в плейлист.

### Settings

- Сократить верхний уровень до 5-6 групп и добавить поиск.
- Скрыть roadmap-пункты из production UI.
- Для мгновенных изменений добавить локальный undo/revert.
- Для Custom CSS/theme import сделать preview, safe mode и timed rollback.
- Заменить ввод списков через запятую на chip editor/add-remove rows.
- Исправить labels, names, autocomplete и focus-visible у полей.
- Реализовать ARIA tabs с roving tabindex и Arrow navigation.

### Equalizer

- Сохранить десятиполосный expert mode, но начать с понятных музыкальных presets.
- Реализовать клавиши Arrow/Home/End и точное значение dB.
- Добавить A/B bypass рядом с графиком.
- Предупреждать о clipping/headroom при положительном gain.
- Разрешить назвать и сохранить свой preset.

### Lyrics и Now Playing

- Развести loading, no lyrics, unsynced lyrics и provider error.
- Дать компактный эквивалент панели при ширине меньше 1200px.
- Уменьшить визуальную конкуренцию lyric card с текущим треком.

### Общесистемные детали

- Увеличить критичные цели 28-36px до 44px там, где позволяет плотность.
- Не скрывать scroll affordance в длинной очереди и Settings полностью.
- Нормализовать API errors в человеческие сообщения с конкретным следующим шагом.
- Добавить shortcut overlay/command palette help.
- Добавить onboarding через первые реальные действия, а не тур слайдов.
